# 智能评估系统需求说明

## 1. 项目概述

### 1.1 背景
随着大语言模型(LLM)和RAG系统的广泛应用，如何科学、系统地评估这些系统的性能成为一个关键挑战。本项目旨在构建一个智能评估系统，提供端到端的评估能力，从测试数据管理、模型配置、指标评估到根因分析，形成完整的评估闭环。

### 1.2 目标
- 提供可视化配置LLM、Embedding、Reranker模型的能力
- 支持多种评估指标对RAG/LLM系统进行全面评估（指标市场可勾选）
- 实现评估结果的根因分析，指导系统调优
- 构建评测数据市场，积累海量高质量评测数据
- 支持测试数据的智能矫正，生成标准答案
- **适配主流开源RAG系统（Dify、Coze、FastGPT、n8n等）**
- **支持从主流RAG系统数据库同步数据（分片、对话记录等）**

### 1.3 目标用户
| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 算法工程师 | RAG/LLM系统开发者 | 快速评估系统性能，定位问题，指导调参 |
| 测试工程师 | 质量保障人员 | 管理测试数据，执行评估任务，生成报告 |
| 产品经理 | 关注产品效果 | 查看评估结果对比，了解系统改进方向 |
| 数据管理员 | 评测数据维护 | 维护评测数据集，管理数据市场 |

## 3. RAG系统适配器架构

### 3.1 设计理念
通过适配器模式统一对接多种RAG系统，实现：
- 统一的查询接口
- 统一的响应格式
- 统一的健康检查
- 支持自定义扩展

### 3.2 支持的RAG系统

| 系统 | 适配器 | 核心API | 特点 |
|-----|--------|--------|------|
| **Dify** | `DifyAdapter` | `/chat-messages`, `/completion-messages` | 支持Chat、Agent、Completion模式，知识库检索 |
| **Coze** | `CozeAdapter` | `/chat` (v3 API) | Bot模式，字节跳动平台，知识库检索 |
| **FastGPT** | `FastGPTAdapter` | `/chat/completions` | 知识库问答，返回引用信息 |
| **n8n** | `N8nAdapter` | Webhook | Workflow模式，灵活配置AI节点 |
| **自定义** | `CustomAdapter` | 自定义URL | 完全自定义请求/响应格式 |

## 4. 评估指标体系

### 4.1 指标来源

| 来源 | 框架 | 指标数量 | 说明 |
|-----|------|---------|------|
| **Ragas** | ragas库 | 7+ | RAG评估主流框架，RAG专用指标 |
| **EvalScope** | evalscope库 | 3+ | 阿里开源，BLEU/ROUGE等 |
| **自研** | internal | N | 语义相似度、响应时间等 |
| **用户贡献** | custom | 无限 | 用户自定义指标插件 |

### 4.2 指标分类

#### 检索质量指标 (Retrieval)
| 指标名称 | 说明 | 需要LLM | 需要Ground Truth | 需要Contexts |
|---------|------|---------|-----------------|-------------|
| Context Precision | 检索内容中与问题相关的比例 | Yes | No | Yes |
| Context Recall | 检索内容覆盖答案的比例 | Yes | Yes | Yes |
| Context Relevancy | 检索内容的相关程度 | Yes | No | Yes |

#### 生成质量指标 (Generation)
| 指标名称 | 说明 | 需要LLM | 需要Ground Truth | 需要Contexts |
|---------|------|---------|-----------------|-------------|
| Faithfulness | 答案对检索内容的忠实度（无幻觉） | Yes | No | Yes |
| Answer Relevancy | 答案与问题的相关性 | Yes | No | No |
| Answer Correctness | 答案的综合准确程度 | Yes | Yes | No |
| Answer Similarity | 答案与标准答案的语义相似度 | No | Yes | No |

#### 质量指标 (Quality)
| 指标名称 | 说明 | 需要LLM | 需要Ground Truth |
|---------|------|---------|-----------------|
| BLEU | n-gram匹配质量 | No | Yes |
| ROUGE-L | 最长公共子序列相似度 | No | Yes |
| Semantic Similarity | Embedding语义相似度 | No | Yes |

#### 性能指标 (Performance)
| 指标名称 | 说明 | 需要LLM |
|---------|------|---------|
| Response Time | 响应时间（越低越好） | No |
| Token Efficiency | Token利用效率 | No |

### 5.2 核心表SQL定义

#### 文档与分片表
```sql
-- 文档表（存储原始文档）
CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(500),
    content TEXT,                    -- 原始文档内容
    file_path VARCHAR(1000),         -- MinIO存储路径
    file_type VARCHAR(50),           -- pdf/txt/docx/html
    source_type VARCHAR(50),         -- upload/huggingface/api
    source_url VARCHAR(1000),
    content_hash VARCHAR(64),        -- 内容去重
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 分片表（存储切分后的分片，向量存储在Milvus中）
CREATE TABLE chunks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    doc_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    chunk_index INT NOT NULL,
    start_char INT,
    end_char INT,
    milvus_id VARCHAR(100),           -- Milvus中的向量ID
    embedding_model VARCHAR(100),     -- 使用的Embedding模型
    embedding_dimension INT,          -- 向量维度
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT NOW(),
    
    UNIQUE(doc_id, chunk_index)
);

CREATE INDEX idx_chunks_doc_id ON chunks(doc_id);
CREATE INDEX idx_chunks_milvus_id ON chunks(milvus_id);
```

#### Milvus Collection设计
```python
# Milvus Collection: chunks
# 向量集合，存储分片的Embedding向量

from pymilvus import Collection, FieldSchema, CollectionSchema, DataType

fields = [
    FieldSchema(name="id", dtype=DataType.VARCHAR, max_length=100, is_primary=True),
    FieldSchema(name="doc_id", dtype=DataType.VARCHAR, max_length=100),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=8000),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=1536),  # 可配置维度
    FieldSchema(name="chunk_index", dtype=DataType.INT64),
    FieldSchema(name="created_at", dtype=DataType.INT64),  # Unix timestamp
]

schema = CollectionSchema(fields=fields, description="文档分片向量集合")
collection = Collection(name="chunks", schema=schema)

# 创建向量索引
index_params = {
    "metric_type": "COSINE",
    "index_type": "IVF_FLAT",
    "params": {"nlist": 1024}
}
collection.create_index(field_name="embedding", index_params=index_params)

### 6.5 同步流程

```
用户操作流程：
Step 1: 选择数据源类型（Dify/FastGPT/n8n/自定义）
Step 2: 填写数据库连接配置
        - 主机地址、端口、数据库名
        - 用户名、密码
Step 3: 测试连接
        - 验证连接是否成功
        - 检测是否为目标系统数据库
Step 4: 选择要同步的表/集合
        - 系统自动列出可用表
        - 显示每个表的记录数
Step 5: 配置字段映射
        - 使用系统默认映射
        - 或自定义字段映射
Step 6: 选择目标数据集
        - 选择已创建的数据集
        - 或创建新数据集
Step 7: 执行同步
        - 全量同步或增量同步
        - 查看同步进度
Step 8: 查看同步结果
        - 同步记录数统计
        - 失败记录详情
```

## 9. 前端交互流程

### 9.1 创建评估任务流程

```
Step 1: 选择数据集
    - 浏览已创建的数据集
    - 查看数据集完整性（是否有ground_truth、contexts）
    - 选择一个数据集

Step 2: 选择RAG系统
    - 选择要评估的RAG系统（已配置好的）
    - 可选择多个系统进行对比评估
    - 系统显示健康状态

Step 3: 进入指标市场勾选指标
    - 按分类筛选：检索/生成/质量/性能/自定义
    - 按标签筛选：RAG必备、防幻觉、快速等
    - 搜索关键词查找
    - 展开查看指标详情和参数
    - 勾选需要的指标组合
    - 调整每个指标的权重和参数

Step 4: 配置评估参数
    - 选择用于评估的LLM模型（部分指标需要）
    - 配置批量处理参数
    - 设置超时时间

Step 5: 检查兼容性
    - 系统自动检查指标依赖
    - 提示缺少的数据字段
    - 提示需要配置的模型

Step 6: 执行评估
    - 提交评估任务
    - 查看实时进度
    - 查看已完成的结果

Step 7: 查看结果
    - 查看各指标得分统计
    - 查看单条记录详细分析
    - 查看根因分析建议
    - 导出评估报告
```

### 9.2 配置RAG系统流程

```
Step 1: 选择系统类型
    - 浏览支持的系统类型：Dify、Coze、FastGPT、n8n、自定义
    - 查看各系统的配置要求

Step 2: 填写连接配置
    - 根据系统类型显示不同的配置表单
    - 配置表单基于connection_schema动态生成
    - API Key等敏感信息加密显示

Step 3: 测试连接
    - 点击"测试连接"按钮
    - 发送健康检查请求
    - 显示连接成功/失败状态

Step 4: 保存配置
    - 配置保存成功
    - 系统进入active状态
    - 可用于评估任务
```

### 9.3 数据同步流程

```
Step 1: 选择数据源类型
    - 浏览支持的系统：Dify、FastGPT、n8n、Coze、自定义数据库
    - 查看每个系统的数据库类型和可同步数据

Step 2: 填写数据库连接配置
    - 输入主机地址、端口、数据库名
    - 输入用户名、密码
    - 选择要同步的表（可选）

Step 3: 测试连接
    - 点击"测试连接"按钮
    - 系统验证连接是否成功
    - 自动检测是否为目标系统数据库

Step 4: 浏览数据库结构
    - 查看所有可用的表/集合
    - 查看每个表的字段结构和记录数
    - 预览表数据（前10条）

Step 5: 配置字段映射
    - 选择目标表
    - 使用系统默认映射（自动识别）
    - 或自定义字段映射：
        - 源字段 → 目标字段
        - 数据转换函数（如extract_contexts）
    - 配置过滤条件（如只同步已完成的分片）

Step 6: 选择目标数据集
    - 选择已创建的数据集
    - 或创建新数据集

Step 7: 执行同步
    - 选择同步类型：全量/增量
    - 点击"开始同步"
    - 查看实时进度条
    - 查看已同步记录数

Step 8: 查看同步结果
    - 同步成功记录数
    - 失败记录数和错误详情
    - 数据预览
    - 可重新同步或增量同步
```
